real key) requires a registered eBird API key, which is not provisioned in
this environment. As an alternative live verification, we exercised the

## Live test path (env-gated — pending eBird key registration)

The full live test (``test_live_bewickwren_over_ca_bbox``) is gated by
``GRACE2_TEST_LIVE_EBIRD=1`` + ``GRACE2_EBIRD_API_KEY=...``. When run with
a real key it:

- Hits ``https://api.ebird.org/v2/data/obs/geo/recent/bewwre?lat=...&lng=...``
  for each tile center in the bbox (-122, 38, -119, 40) cover.
- Dedupes by ``subId`` across tiles.
- Hard-filters to keep only points strictly inside the requested bbox
  (job-0086 codified lesson).
- Writes evidence to ``services/agent/evidence/ebird_live_job_0128.txt``.

This test is marked ``pytest.mark.skipif`` so it does NOT fail the unit
suite when the key is absent — per audit.md "do NOT fail unit suite if
key is missing".
