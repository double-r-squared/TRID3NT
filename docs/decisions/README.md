# docs/decisions/

This directory holds one-page resolution documents for Open Questions enumerated in SRS §6. Each file closes a specific OQ by recording context, options considered, the decision reached, downstream consequences, and references — creating a permanent, auditable record of why a given architectural choice was made, separate from the SRS prose itself (which is amended by the orchestrator at sprint close, per the v0.3.15 OQ-5 closure pattern). The first document (`oq-4-hydromt-depth.md`, landed in sprint-07 by job-0038-engine-20260606) establishes the format; all future OQ resolutions follow the same five-section structure (`Context`, `Options Considered`, `Decision`, `Consequences`, `References`) and the same constraints: one file per OQ, title matching `# OQ-<n>: <decision title>`, under 300 lines, with a concrete single-option recommendation and at least the three minimum open sub-questions required by the kickoff. Downstream jobs consuming a decision should cite its filename and the relevant section rather than re-litigating the tradeoffs.
