## Domain Discipline

- **OpenTofu from job one.** No console-clicked resources that IaC doesn't capture. If you click anything in the AWS console to unblock yourself, immediately import it into state or recreate it through IaC and delete the manual artifact. A resource that exists but isn't in code is debt, not infrastructure. The live modules use the `grace2-tofu-state-226996537797` S3 backend.
- **AWS auth is the user's step.** AWS-mutating commands (`tofu apply`, `aws ec2 modify-instance-attribute`, IAM changes, ECR pushes) run under NATE's credentials only â€” no agent or CI system runs them. Surface `aws configure` / SSO login and any `tofu apply` as a manual user action in your report and runbook; never script around it or fake credentials. IAM/login is always NATE's step.
- **Scale to zero by default; always-on only where latency demands it.** AWS Batch solver jobs are Spot + scale-to-zero (NFR-C-2). The agent box auto-stops when idle (waking on the overlay). The only deliberately always-on box is the tiny TiTiler tiles box â€” justified because the map must serve 24/7 even while the agent sleeps. Every always-on dollar is a line in the cost itemization; if it isn't justified by a latency NFR or the map-availability requirement, it shouldn't exist.
- **Heavy/sandboxed compute runs externally per the execution-architecture norm.** Minimal always-on agent + telemetry on every tool/job + external per-job/scale-to-zero execution for heavy work: SFINCS + SWMM on AWS Batch (Spot); MODFLOW + the Python sandbox via local subprocess on the EC2 box (the AWS local-exec path; the GCP Cloud Run Job sandbox is gone but `infra/python-sandbox/executor.py` is load-bearing â€” the local subprocess runs it). This is the default for all new heavy compute.
- **The cancellation chain must terminate within 30s.** Provision Batch so a cancel call (issued by `agent` with the `ExecutionHandle`'s AWS Batch job id, a field `schema` owns) actually stops an in-flight job and frees the Spot capacity. Your job-definition + the agent role's `batch:CancelJob`/`batch:TerminateJob` permission are the third party citing that same handle (Invariant 8, FR-CE-2, FR-AS-6, NFR-R-3).
- **WSS in production, WS only local.** TLS 1.2+ terminates at CloudFront in front of the agent WebSocket; the 1 MB message cap and single-connection-per-session model from Appendix A are properties the ingress must not break (no buffering proxy that chunks frames or strips the upgrade). Disconnect-resilience matters: the agent box auto-stop/wake cycle drops sockets, so clients reconnect automatically and a rendered per-Case layer stays rendered across the cycle.
- **DynamoDB sized to the D.8 baseline, not aspirations.** On-demand capacity fits the v0.1 corpus (1000 articles / 200 events / 100 runs / 50 sessions / 50 projects). The demo runs file-backed; flip to DynamoDB (`GRACE2_PERSISTENCE_BACKEND=dynamodb`) for durability. Secrets (API keys) live in SSM Parameter Store, reached via the instance role â€” never in IaC variables committed to the repo.
- **Inspect images before pushing to ECR (container hygiene).** BEFORE pushing any image to ECR, verify it is bare-bones (size + `docker history` layers + build context), multi-stage where it helps, minimal base, `.dockerignore` present. Report size/layer breakdown before any push. Applies to the SWMM worker, SFINCS worker, cht deck-builder, and the QGIS worker (job-0308).
- **Every AWS resource is tagged.** A tag scheme (`project`, `env`, `sprint`) that lets the cost breakdown be produced mechanically per resource. An untagged resource can't be itemized and fails cost review.
- **Secrets never land in code, repo, or images.** No API keys in Dockerfiles, env files committed to git, or OpenTofu `.tfvars` in the repo. Use SSM Parameter Store + the instance role; verify images contain no baked credentials before push. Strip GCP-era ENV defaults (`GCP_PROJECT`, `grace-2-hazard-prod-*` buckets) from worker Dockerfiles even though Batch `containerOverrides` make them inert â€” honesty over inert cruft.
- **OSI license detectable by GitHub.** The license file lives at repo root in a form GitHub's license detection recognizes (e.g. `LICENSE`), and third-party dependencies are tracked license-compatibly (NFR-L-1/2) â€” this is dependency tracking, not vendoring source. Note the GPL-isolated cht_sfincs deck-builder is a separate Batch image precisely to keep the license boundary clean.
