## Decisions Made

- **Decision:** Playwright 1.49.0 (not floating "latest", not 1.50+).
  - **Rationale:** Pinned to a known-good release; latest minor at install time. Matches the kickoff's "latest stable" guidance without floating the pin. Reproducibility per Linux-everywhere prod posture.
  - **Alternatives considered:** Floating `^1.49.0` (rejected — explicit pin is clearer); Puppeteer (banned by kickoff — Playwright only).

- **Decision:** Two projects — `chromium` + `firefox-esr` — using Playwright's `Desktop Chrome` / `Desktop Firefox` device descriptors with viewport overridden to 1440x900.
  - **Rationale:** Mirrors job-0016 AC4 cross-browser parity (Chromium 148 + Firefox-ESR 140); Playwright tracks both via the same install step. `Desktop Firefox` is the closest match to Firefox-ESR on the Debian dev box that Playwright ships (Playwright does not have a separate ESR channel — see OQ-W-27-2).
  - **Alternatives considered:** Single Chromium project (rejected — NFR-PO-1 requires cross-browser); add WebKit/Safari (deferred — SRS Safari spot-check is post-M3, no macOS substrate; see OQ-W-27-3).

- **Decision:** Resolve `@playwright/test` via explicit relative path inside `tools/screenshot.mjs`, not via bare specifier + `NODE_PATH`.
  - **Rationale:** First attempt used a bare ESM import; Node ESM's resolution walks from the importing file's directory upward, and `tools/` has no ancestor `node_modules/` with `@playwright/test`. Setting `NODE_PATH=web/node_modules` does not work for ESM (Node CommonJS-only feature; verified live). The script-resident relative path is robust to cwd and surfaces a clear error if Playwright isn't installed.
  - **Alternatives considered:** Symlink `tools/node_modules → ../web/node_modules` (rejected — surprising side-effect, leaks in tarballs); root `package.json` workspace (rejected — kickoff explicitly scopes the devDep to `web/package.json`); inline `dynamic import()` of an absolute resolved path (chosen — current implementation).

- **Decision:** `--wait=1500` default (MapLibre first WebGL paint).
  - **Rationale:** Headless Chromium/Firefox don't have a stable "WebGL painted" event; a fixed 1500 ms wait reliably catches first paint on the deployed substrate. Job-0028 acceptance tests can use Playwright's `page.waitForSelector('[data-testid="grace2-map"] canvas')` instead of relying on this CLI wait.
  - **Alternatives considered:** `requestAnimationFrame`-double-tick (rejected — not robust against CORS-blocked tile timeouts); `networkidle` (rejected — never reaches idle when CORS errors retry; see OQ-W-27-1).

- **Decision:** Ship six-state tour even though states 3–5 fall back to initial captures on the current main.
  - **Rationale:** Job-0026 (PipelineStrip) lands after this job; shipping the orchestration scaffold now means the next state captures cost zero infra work — just adding selectors. The fallback is logged loudly (`[screenshot] ... falling back to initial`).
  - **Alternatives considered:** Two-state tour — rejected; the kickoff explicitly names six states.

- **Decision:** Promote only `chromium-initial.png` + `firefox-initial.png` into `tests/m3/artifacts/`, not all twelve.
  - **Rationale:** Kickoff STEP 7 explicitly: "the only 2 states reliably reproducible right now — 0026 and 0025 will provide more". Other ten captures fall back to initial-equivalent frames until 0026 lands; promoting them now would baseline duplicates against a future-different reality.
  - **Alternatives considered:** Promote all twelve (rejected per kickoff); promote zero (rejected — no live-E2E baseline to anchor visual regression).

- **Decision:** `tests/m3/artifacts/screenshots-*/` gitignore pattern (not `tests/m3/artifacts/`).
  - **Rationale:** Lets canonical PNGs sit at `tests/m3/artifacts/*.png` (tracked) while per-run captures land under a date-stamped subdir (gitignored). Avoids fragile `!`-negation against a directory-wide ignore.
  - **Alternatives considered:** `tests/m3/artifacts/` + explicit `!`-negations for each canonical file (rejected — fragile; every new canonical file needs another line); commit nothing (rejected — no visual baseline).
